DATA SOURCES
================================================================================

PRIMARY: Vivli Data Request API
  - Real-time request status
  - Request metadata
  - Form submission data
  - Timeline information

SECONDARY: Knowledge Base
  - Process information
  - Policy details
  - Approval criteria

TERTIARY: Request History
  - Previous interactions
  - Change log
  - Notes and comments

================================================================================
QUERY TYPES HANDLED
================================================================================

TYPE_1: Status queries
  Example: "What's the status of my request?"
  Answer: Current stage + timeline

TYPE_2: Specific field queries
  Example: "Can I modify my data request?"
  Answer: Policy-based response + process steps

TYPE_3: Timeline queries
  Example: "How long does form-check take?"
  Answer: Expected timeline + SLA information

TYPE_4: Access queries
  Example: "When will I get access to the data?"
  Answer: Process timeline + next steps

================================================================================
VALIDATION RULES
================================================================================

RULE_1: Extract request ID from query or user context
RULE_2: Verify request ID exists and is valid
RULE_3: Verify user owns or can access the request
RULE_4: Fetch fresh data from API (no cached stale data)
RULE_5: Verify data freshness (< 5 minutes old)

================================================================================
ERROR MESSAGES
================================================================================

ERROR_1_TYPE: REQUEST_NOT_FOUND
ERROR_1_MESSAGE: "Unable to find that data request. Please verify the request ID."
ERROR_1_WHEN: Invalid or non-existent request ID

ERROR_2_TYPE: PERMISSION_DENIED
ERROR_2_MESSAGE: "You don't have access to that request."
ERROR_2_WHEN: User not owner/authorized

ERROR_3_TYPE: API_ERROR
ERROR_3_MESSAGE: "Unable to retrieve request information. Please try again."
ERROR_3_WHEN: API failure

ERROR_4_TYPE: STALE_DATA
ERROR_4_MESSAGE: "Unable to get current status. Please try again in a moment."
ERROR_4_WHEN: Data more than 5 minutes old

ERROR_5_TYPE: RATE_LIMIT
ERROR_5_MESSAGE: "Too many requests. Please wait a moment and try again."
ERROR_5_WHEN: API rate limit exceeded
