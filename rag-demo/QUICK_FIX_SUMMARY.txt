RAG CHATBOT - PROBLEMS FOUND & FIXED
================================================================================

PROBLEM:
--------
Whatever query users sent, they got: "I couldn't find a reliable answer..."
(Even though documents were indexed!)

Example:
  Query: "what are the things you can answer"
  Response: Escalation message (WRONG!)
  Available docs: 5 documents with relevance score 38+ (IGNORED!)

ROOT CAUSES:
============

1. OVERLY STRICT THRESHOLDS (config.py)
   - CONFIDENCE_THRESHOLD: 0.6 → FIXED: 0.3
   - INTENT_HIGH_CONFIDENCE: 0.25 → FIXED: 0.15

   Why: Keyword search gives lower scores than expected
   Impact: Documents filtered out before use

2. BRITTLE INTENT CLASSIFIER (intent_classifier.py)
   - Required exact keyword matches
   - Query "what are the things you can answer" has no FAQ keywords
   - Result: Marked as UNKNOWN/HYBRID with low confidence

   FIXED: Added baseline score (0.12) for meaningful queries
   Why: Real questions should be recognized even without keywords

3. CONFIDENCE GATE IN HYBRID HANDLER (main.py)
   - Code: if documents and confidence > 0.3:
   - Problem: confidence=0.19, threshold=0.3
   - Result: Documents retrieved but IGNORED

   FIXED: Changed to: if documents:
   Why: If documents exist, use them! Don't gate with confidence

4. CONTEXT TOO LARGE (retrieval.py)
   - format_for_llm() included all 18MB JSON + all 380-row CSVs
   - Total context: 54MB
   - LLM error: "string too long" (max 10MB)

   FIXED: Added truncation (2000 chars per doc, 500KB total)
   Why: LLM can't handle massive documents
